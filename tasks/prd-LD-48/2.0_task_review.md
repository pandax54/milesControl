### 🟢 Minor Issues

**1. Negative prefix character mismatch**
- **File**: `src/components/dashboard/net-value-badge.tsx`, line 31
- **Description**: The task spec (`2_task.md`) specifies the negative prefix as `−` (Unicode minus sign U+2212), but the implementation uses `-` (ASCII hyphen-minus U+002D). The test at line 33 in `net-value-badge.test.tsx` also asserts `-` (hyphen), making tests consistent with implementation but inconsistent with the spec.
- **Impact**: Purely cosmetic — both characters look almost identical, but screen readers and copy-paste behavior may differ.
- **Suggestion**: Align with the spec by using `−` (Unicode minus) in `NET_VALUE_BADGE_CONFIG.negative.prefix` and updating the test assertion to match.

```ts
// net-value-badge.tsx line 31
prefix: "−",   // Unicode minus sign, not ASCII hyphen
```

**2. Missing edge-case test for `netValue = 0`**
- **File**: `src/components/dashboard/net-value-badge.test.tsx`
- **Description**: The task implementation details explicitly call out "netValue is exactly 0 → show as neutral with '~R$0.00'" as an edge case to handle. The component handles it correctly (passes `Math.abs(0)` to `formatCurrency`), but there is no corresponding test asserting this behavior.
- **Suggestion**: Add a dedicated test case:

```tsx
it("should render a neutral badge with tilde prefix when netValue is exactly 0", () => {
  render(<NetValueBadge netValue={0} netValueType="neutral" />);
  const badge = screen.getByRole("generic", { name: /neutral/i });
  expect(removeWhitespace(badge.textContent ?? "")).toBe(removeWhitespace(`~${formatCurrency(0)}`));
});
```
