### 🟢 Minor Issues

1. **`src/lib/services/promotion.service.ts:325`** — `getPromotionById` declares return type `Promise<Promotion | null>` but the query includes `sourceProgram` and `destProgram` relations via `include`, so the actual runtime return is `PromotionWithPrograms | null`. The declared type hides the included relations from callers. Should be `Promise<PromotionWithPrograms | null>` for accuracy. Low impact since this function is not consumed yet.

2. **`src/components/promotions/promotion-feed.tsx:51`** — The `hasFilters` check for the empty state inspects `type`, `programId`, and whether `status` differs from default, but "Active" is the default status. This means selecting "Active" explicitly still counts as "no filter" for empty-state messaging. Behavior is reasonable but slightly inconsistent. Non-blocking.

3. **Future consideration: pagination** — The feed fetches up to `DEFAULT_LIST_LIMIT` (50) promotions. As the promotion database grows, adding cursor-based pagination or "Load More" would improve performance. Not a blocker for this task.
