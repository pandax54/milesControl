### 🟢 Minor Issues

1. **Sequential user processing in `sendAllDigests`** — `balance-digest.service.ts:227` — Users are processed with a sequential `for` loop. For large user bases this could be slow. Consider batching with `Promise.allSettled` and a concurrency limiter (e.g., p-limit) in a future optimization pass. Acceptable for current scale.

2. **`programName` denormalization without `Program` relation** — `schema.prisma:493` — The `BalanceSnapshot` model stores `programName` as a string without a `@relation` to `Program`. If a program is renamed, historical snapshots show the old name. This is an acceptable trade-off for query simplicity and correct historical context.

3. **No unsubscribe link in email** — `balance-digest.service.ts:108-134` — The digest email does not include an unsubscribe mechanism. Required for compliance in many jurisdictions. Plan for Phase 4 (Alerts & Notifications).

## ✅ Positive Highlights

1. **Excellent separation of concerns** — Pure `computeBalanceChanges()` and `buildDigestHtml()` functions are trivially testable with no side effects.

2. **Graceful degradation** — Resend integration returns `false` when API key is missing rather than throwing, allowing the system to work without email in development.

3. **Security-conscious cron auth** — Bearer token verification with explicit `IS_DEVELOPMENT` gate for the fallback path, preventing unauthenticated access in production.

4. **Typed mock factories** — All test files use properly typed `buildMockEnrollment()`, `buildMockSnapshot()`, `buildMockUser()` factories with correct Prisma types. No `as never` casts.

5. **Comprehensive test coverage** — 30 tests across service (24) and cron route (6) covering happy paths, error paths, edge cases (empty enrollments, email failures, multi-snapshot deduplication), and security (auth header validation, dev vs prod fallback).

6. **Named color constants** — `COLOR_POSITIVE`, `COLOR_NEGATIVE`, `COLOR_NEUTRAL` replace magic hex values.

7. **Portuguese-first UX** — Email content is entirely in Portuguese with proper greetings, summaries, and footer appropriate for the Brazilian target audience.

8. **Snapshot deduplication logic** — Correctly keeps the earliest snapshot per program by leveraging desc ordering and Map overwrite semantics, with explicit test verifying this behavior.

9. **Error isolation in batch processing** — `sendAllDigests` catches per-user errors and continues processing remaining users, preventing one failure from blocking the entire digest run.

10. **PII-safe logging** — Email addresses are not logged; only userId and aggregate counts appear in log output.

11. **Zod-validated env config** — New env vars (`RESEND_API_KEY`, `RESEND_FROM_EMAIL`, `CRON_SECRET`) added to centralized schema with sensible defaults.
