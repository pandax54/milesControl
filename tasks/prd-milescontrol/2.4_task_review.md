### 🟢 Minor Issues

1. **`src/actions/credit-cards.test.ts:65,69`** — `as never` casts for mock return values (`mockAuth.mockResolvedValue(MOCK_SESSION as never)`, `mockCreateCreditCard.mockResolvedValue({} as never)`). The service test file was properly updated with a typed `buildMockCard()` factory, but the action tests still use `as never` for session and service return mocks. Consider typing these more precisely.

2. **`src/components/dashboard/credit-card-form-fields.tsx:167-169`** — The "Add" button retrieves the benefit input value via DOM traversal (`e.currentTarget.previousElementSibling as HTMLInputElement`). This is fragile — if the DOM structure changes, it breaks silently. Consider using a `useRef` or controlled state for the benefit input instead.

3. **`src/components/dashboard/credit-card-form-fields.tsx:121-128`** — Uses a native `<input type="checkbox">` instead of the shadcn/ui Checkbox component, creating a minor visual inconsistency with the rest of the design system.

## ✅ Positive Highlights

- **Shared form fields component** (`credit-card-form-fields.tsx`): Clean extraction of form logic shared between create and edit dialogs, with `parseAndValidateForm` and `isFormComplete` helpers — eliminates ~80% form code duplication.
- **Decimal normalization at page boundary**: `Number(card.annualFee)` conversion in the server component keeps Prisma `Decimal` types out of the UI layer.
- **Edit dialog resets on open**: `handleOpenChange` rebuilds form values from the card prop each time the dialog opens, preventing stale state.
- **Ownership checks in service layer**: Both `updateCreditCard` and `deleteCreditCard` verify the card belongs to the requesting user before mutating.
- **Custom error class** (`CreditCardNotFoundError`): Clean domain error with proper handling at the action boundary.
- **Comprehensive Zod schemas**: All three operations have proper validation with nullable/optional handling for clearable fields like `pointsPerDollar` and `benefits`.
- **Type-safe test factory** (`buildMockCard`): Properly typed with `Prisma.Decimal` for `annualFee` — addresses the prior `as never` issue in service tests.
- **Partial updates**: Service correctly builds a sparse data object using spread + undefined checks, so only changed fields are sent to Prisma.
- **Prisma.JsonNull handling**: Benefits and nullable JSON fields correctly use `Prisma.JsonNull` instead of raw `null`.
- **Thorough test coverage**: 17 schema + 14 service + 15 action = 46 tests for this feature alone. Coverage: 97.19% stmts, 91% branches.
- **Conditional top-level "Add Card" button**: Only renders when cards exist, avoiding visual redundancy with the empty state CTA.
- **Pino logging**: All service mutations log structured context (`userId`, `cardId`) with data-first syntax. No `console.log` anywhere.
- **Cards grouped by bank**: Clean `reduce` + sort pattern in the page component for organized display.
