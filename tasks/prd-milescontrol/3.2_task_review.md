3. **Clean re-export removal** — Removed backward-compatibility re-export from PdP scraper. PdP tests now import PdP-specific helpers directly, and shared helpers import from `./promotion-helpers`.

4. **Dedicated `promotion-helpers.test.ts`** — 26 shared helper tests moved from PdP test file to their own file for clear ownership and coverage attribution. No tests lost in the refactor.

5. **Brazilian date parsing** — `parseBrDate` correctly handles "DD/MM/YYYY as HH:MM" format with proper validation (`Number.isNaN` check), graceful fallbacks for missing dates, and embedded-in-text extraction.

7. **Comprehensive test coverage** — 32 MC-specific tests covering: `extractCategory` (4), `extractDateText` (3), `parseBrDate` (7), and full integration tests (18) including edge cases for missing titles, dates, categories, external links (melhoresdestinos.com.br), empty pages, multiple articles, and rawContent.

8. **Consistent scraper architecture** — MC scraper follows the exact same pattern as PdP: constructor with `Partial<ScraperConfig>` overrides, `extractPromotions` override, site-specific helpers exported and individually tested.

9. **Pure functions** — All extraction helpers (`extractCategory`, `extractDateText`, `parseBrDate`) are pure, individually exported, and testable.

10. **No `any` types, no `console.log`, no `process.env` access** — Clean TypeScript throughout. Pino logging with structured data (`{ articlesFound }`).
