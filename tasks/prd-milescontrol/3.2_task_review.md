## Summary

Task 3.2 implements the Melhores Cartoes scraper (`MelhoresCartoesScraper`) that scrapes promotions from `melhorescartoes.com.br/c/promocoes-milhas`. The implementation follows the established scraper architecture from tasks 3.0 and 3.1 cleanly.

A key architectural decision was extracting shared promotion helpers (`classifyPromoType`, `extractBonusPercent`, `extractDiscountPercent`, `extractPrograms`, and related constants/patterns) from the PdP scraper into a new `promotion-helpers.ts` module. The PdP module re-exports these for backward compatibility, and all 641 existing tests continue to pass.

The new scraper correctly parses `div.post-promo-home-container` elements, extracts titles from `h3 > a`, categories from `.label-card`, and dates from `.date` or `.fim-card` elements. Brazilian date format `DD/MM/YYYY as HH:MM` is handled with a dedicated parser. The implementation is clean, well-structured, and follows project conventions.

### Minor Issues

**1. Unused `$: CheerioAPI` parameter in `extractCategory` and `extractDateText`**
- File: `src/lib/scrapers/melhores-cartoes.ts`, lines 81 and 87
- Both helper functions accept `$: CheerioAPI` as their first parameter but never use it. Only the `container` parameter is used via `container.find(...)`. This is consistent with the PdP scraper pattern (`extractCategories` also accepts `$`), so it may be intentional for API consistency. However, unused parameters are a minor code smell.
- Suggested fix: Either remove the `$` parameter if it will never be needed, or prefix with underscore (`_$`) to signal intentional non-use. Since the PdP scraper has the same pattern, this could be addressed across both scrapers together.

**2. Missing dedicated test file for `promotion-helpers.ts`**
- File: `src/lib/scrapers/promotion-helpers.ts`
- The shared helper module has no co-located `promotion-helpers.test.ts` file. The functions are tested indirectly through the PdP test file (via re-exports). Per project convention (tests.md rule: "Co-located test files"), this module should have its own test file.
- This is classified as minor rather than major because: (a) the functions are thoroughly tested through the existing PdP tests (all pass), and (b) moving the tests now would be a larger refactoring effort better suited for a dedicated cleanup task. However, as more scrapers import directly from `promotion-helpers.ts`, having co-located tests becomes increasingly important.

**3. Comment reads "Try label-card span first, then fall back to any element with label-card class" but function only does one thing**
- File: `src/lib/scrapers/melhores-cartoes.ts`, line 82
- The comment in `extractCategory` describes a two-step fallback strategy, but the implementation only has one step: `container.find('.label-card').first()`. The fallback described in the comment is not implemented. The comment should match the actual behavior.
- Suggested fix: Update the comment to simply say `// Extract category from the first .label-card element` or remove it entirely (the code is self-explanatory).

**4. `parseBrDate` constructs dates via string parsing which may have timezone ambiguity**
- File: `src/lib/scrapers/melhores-cartoes.ts`, line 118
- The date string `new Date("2026-03-20T15:30:00")` is parsed without a timezone offset. The `Date` constructor interprets this as local time in most engines, but the behavior can vary. Since the source site is Brazilian, the dates are implicitly BRT/BRST. This is acceptable for `detectedAt` timestamps (approximate is fine), but worth noting if precise timezone handling becomes important later.

## Recommendations

2. **Consider creating `promotion-helpers.test.ts`** in a future cleanup task: Move the shared helper tests (`classifyPromoType`, `extractBonusPercent`, `extractDiscountPercent`, `extractPrograms`) from the PdP test file to a dedicated co-located test file. 3. **Address unused `$` parameter**: Either remove it from `extractCategory` and `extractDateText` signatures, or prefix with underscore. If keeping for API consistency with the PdP scraper, consider updating both scrapers together.
