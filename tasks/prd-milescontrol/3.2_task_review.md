### 🟢 Minor Issues

1. **Duplicated `CheerioSelection` type alias** — `melhores-cartoes.ts:79` and `passageiro-de-primeira.ts:92` both define `type CheerioSelection = ReturnType<CheerioAPI>`. Consider extracting to `promotion-helpers.ts` or a shared scraper types file so future scrapers (3.3, 3.4) reuse it instead of redefining it.

2. **Re-export coupling in PdP** — `passageiro-de-primeira.ts:12` re-exports shared helpers for backward compatibility with existing PdP tests. This works but couples PdP tests to the PdP module for helpers that now live in `promotion-helpers.ts`. Consider updating PdP test imports to import directly from `./promotion-helpers` and removing the re-export line.

3. **No dedicated `promotion-helpers.test.ts`** — The shared helpers (classifyPromoType, extractBonusPercent, extractDiscountPercent, extractPrograms) are tested through the PdP test file (via re-exports). Coverage is 100% on `promotion-helpers.ts`, so this is non-blocking. However, a dedicated test file would make ownership clearer as more scrapers are added.

4. **`parseBrDate` timezone ambiguity** — `melhores-cartoes.ts:117` constructs dates via `new Date("YYYY-MM-DDTHH:MM:SS")` without a timezone offset. The `Date` constructor interprets this as local time, which is acceptable for `detectedAt` timestamps (approximate is fine) but worth noting if precise timezone handling becomes important later.
