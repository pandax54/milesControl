## Summary

Task 3.5 implements the promotion deduplication and storage service that bridges scraped promotions (from tasks 3.0–3.4) into the database. The service resolves scraped program names to database IDs via case-insensitive lookup, deduplicates promotions via same-source URL matching (backed by a `@unique` schema constraint) and cross-source heuristic matching (type + programs + bonus/discount within a 30-day window), computes ACTIVE/EXPIRED status based on deadlines, and provides query functions for listing and retrieving promotions. The implementation is clean, well-structured, and thoroughly tested with 34 tests across all exported functions.

## ✅ Positive Highlights

1. **Clean two-tier deduplication strategy**: Same-source URL match updates existing records; cross-source heuristic match (type + programs + bonus/discount within 30-day window) correctly skips duplicates from different blogs covering the same promotion.
2. **Schema-level integrity**: `@unique` on `sourceUrl` enforces dedup at the database level, correctly backing the `findUnique` call in `findBySourceUrl`.
3. **Per-item error handling**: Each promotion in the batch is wrapped in try/catch with a `failed` counter, ensuring one bad promotion doesn't abort the entire batch.
4. **Resilient ScraperRun update**: The `scraperRunId` update is wrapped in try/catch so a failed side-effect doesn't discard storage results, with a dedicated test covering this path.
5. **Comprehensive test coverage**: 34 tests cover all exported functions with edge cases (empty arrays, null programs, unresolvable names, mixed batch outcomes, error paths, deadline status variations, optional field nullification, ScraperRun failure).
6. **Deterministic date testing**: Tests use `vi.useFakeTimers()` / `vi.setSystemTime()` for reliable date comparisons, with `vi.useRealTimers()` cleanup in `afterEach`.
7. **Proper Pino logging**: All log calls use structured data-first pattern (`logger.info({ ... }, "message")`), error objects as `err` property, appropriate levels (debug for dedup skips, info for results, error for failures). No `console.log` anywhere.
8. **Named constants**: `CROSS_SOURCE_DEDUP_WINDOW_DAYS` and `DEFAULT_LIST_LIMIT` eliminate magic numbers.
9. **`readonly` on interfaces**: `StorePromotionsResult` properties are all `readonly`; input uses `readonly ScrapedPromotion[]`.
10. **Case-insensitive program resolution**: `mode: 'insensitive'` in `resolveProgramId` handles inconsistent casing from different scraper sources.
11. **No `as never` casts**: Previous review's `as never` workarounds have been fully removed — mock typing is now clean.
