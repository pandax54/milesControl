### 🔴 Critical Issues

**1. Unreachable dead code in `callSeatsAeroApi` function**

**File**: `src/lib/integrations/seats-aero.ts`
**Lines**: 161
**Severity**: CRITICAL
**Description**:

The line `throw new SeatsAeroError('All retry attempts exhausted');` on line 161 is unreachable. The retry loop (lines 121-158) will always exit via explicit `throw` statements on lines 144 or 148 before the loop completes naturally. Specifically:

- When `isLastAttempt === true` (line 140), the code throws the original error on line 148
- When an error occurs on iteration 3 of MAX_RETRY_ATTEMPTS, `isLastAttempt` evaluates to true and the error is thrown
- The loop variable `attempt` is incremented in the for loop header (line 121), so after the 3rd iteration, the loop condition fails and execution exits the loop
- However, before that happens, on the 3rd iteration, `isLastAttempt === true` causes a throw on line 148

This dead code indicates incomplete error handling logic.

**Suggested Fix**:

Replace the unreachable throw statement with a proper error. The retry logic should ensure that all attempts genuinely fail before reaching this point. Consider one of these approaches:

**Option A**: Remove the dead code and ensure the loop properly exhausts retries
```typescript
async function callSeatsAeroApi<T>(
  endpoint: string,
  queryParams?: Record<string, string>,
): Promise<T> {
  const apiKey = env.SEATS_AERO_API_KEY;

  if (!apiKey) {
    throw new SeatsAeroError('Seats.aero API key not configured');
  }

  const url = new URL(`${SEATS_AERO_BASE_URL}${endpoint}`);

  if (queryParams) {
    for (const [key, value] of Object.entries(queryParams)) {
      url.searchParams.set(key, value);
    }
  }

  let lastError: Error | unknown;

  for (let attempt = 1; attempt <= MAX_RETRY_ATTEMPTS; attempt++) {
    try {
      const response = await fetch(url.toString(), {
        headers: {
          'Partner-Authorization': apiKey,
          'Content-Type': 'application/json',
        },
      });

      if (!response.ok) {
        const errorText = await response.text();
        throw new SeatsAeroApiError(
          `Seats.aero API returned ${response.status}: ${errorText}`,
          response.status,
        );
      }

      return (await response.json()) as T;
    } catch (error) {
      lastError = error;
      const isLastAttempt = attempt === MAX_RETRY_ATTEMPTS;

      if (error instanceof SeatsAeroApiError && error.statusCode < 500) {
        // Client errors (4xx) are not retryable
        throw error;
      }

      if (isLastAttempt) {
        // All attempts exhausted - rethrow with context
        throw lastError;
      }

      const delayMs = BASE_RETRY_DELAY_MS * Math.pow(2, attempt - 1);
      logger.warn(
        { endpoint, attempt, delayMs, err: error },
        'Seats.aero API call failed, retrying',
      );

      await sleep(delayMs);
    }
  }

  // This line is also unreachable but serves as a type-safe fallback
  throw new SeatsAeroError('All retry attempts exhausted');
}
```

This ensures the code path is explicit and the error is thrown with context about which error occurred on the final attempt.

### 🟡 Major Issues

**1. Missing Zod validation for API responses**

**File**: `src/lib/integrations/seats-aero.ts`
**Lines**: 138 (in `callSeatsAeroApi`), 198, 209, 212 (in `mapAvailabilityToAwardFlights`)
**Severity**: MAJOR
**Description**:

The API response is parsed without validation:
- Line 138: `return (await response.json()) as T;` - Uses generic type assertion without runtime validation
- Lines 198, 209, 212: Use `as number` type assertions without verifying that the values are actually numbers

Per project standards (rule: code-standards.md), "Never use `any`; use `unknown` with type guards". While these are type assertions (not `any`), they bypass the Zod validation standard established in the codebase. The env configuration uses Zod validation, and all external API responses should be validated similarly.

**Example of the issue**:

```typescript
// Current: No validation
const milesRequired = availability[mileageKey] as number; // Could be anything!
if (milesRequired <= 0) {
  return null;
}

// Safe: With validation
const mileageValue = availability[mileageKey];
if (typeof mileageValue !== 'number' || mileageValue <= 0) {
  return null;
}
```

Create Zod schemas for the Seats.aero API responses in `src/lib/validators/seats-aero.schema.ts`:
