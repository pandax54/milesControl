## Key Insights

### VAPID Setup Pattern

The module-level configuration pattern is excellent:

```typescript
let vapidConfigured = false;

function ensureVapidConfigured(): boolean {
  if (vapidConfigured) return true;  // Already configured
  if (!env.VAPID_PUBLIC_KEY || !env.VAPID_PRIVATE_KEY) {
    logger.warn('VAPID keys not configured');
    return false;  // Graceful degradation
  }
  webpush.setVapidDetails(...);
  vapidConfigured = true;
  return true;
}
```

Benefits:
- Called once per module load
- Prevents redundant setup overhead
- Explicit graceful degradation
- Testable with `vi.resetModules()` pattern
