### 🟢 Minor Issues

1. **Missing `aria-label` on external link icon button** — `dashboard-balances.tsx:50-58` renders an `<a>` with only an `<ExternalLink>` icon and no accessible text or `aria-label`. Screen readers cannot convey the link's purpose.

   **Suggested fix**: Add `aria-label={`Visit ${enrollment.program.name} website`}` to the `<a>` element.

2. **Rating thresholds duplicated (tracked for task 3.6)** — `dashboard-transfers.tsx:18-20` defines `COST_THRESHOLD_EXCELLENT = 12`, `COST_THRESHOLD_GOOD = 16`, `COST_THRESHOLD_ACCEPTABLE = 20` with a `rateTransfer()` function. These match the PRD domain thresholds (F2.4) and are already named constants (good). Extraction to shared `src/lib/utils/cost-rating.ts` is already deferred to task 3.6 per prior review.

3. **`date-fns` imported only in `dashboard-balances.tsx`** — The `formatDistanceToNow` function is the sole `date-fns` usage across the project. Low priority — `date-fns` is tree-shakeable. Consider native `Intl.RelativeTimeFormat` if no other components adopt it.

## ✅ Positive Highlights

- **Parallel data fetching** — `fetchDashboardData` uses `Promise.all` for 4 independent queries (enrollments, subscriptions, transfers, projections), minimizing latency.
- **Decimal-to-number conversion at the service boundary** — `monthlyCost`, `totalCost`, and `costPerMilheiro` are all converted via `Number()` in the service layer before reaching UI.
- **Clean component decomposition** — 6 focused components, each with a single responsibility, clear props interface, and proper TypeScript typing. The page is a thin orchestrator.
- **Empty state handling** — Both the page level (no enrollments → onboarding CTA) and individual sections (no subscriptions, no transfers, no projections) have clear, helpful empty states with navigation links.
- **Staleness classification** — `classifyStaleness` is a pure, exported function with injected `now` parameter for testability. 6 boundary tests cover all edge cases including exact threshold values.
- **Readonly types throughout** — All interfaces use `readonly` properties and `readonly` arrays, enforcing immutability from service through to UI.
- **Dark mode support** — Staleness colors and alert banner include both light and dark mode classes.
- **Typed mock factories in tests** — `buildMockEnrollment`, `buildMockSubscription`, `buildMockTransfer` use proper typed interfaces with `Prisma.Decimal` — no `as never` casts.
- **Shared format utilities** — `formatNumber` and `formatCurrency` in `src/lib/utils/format.ts` with 9 tests covering edge cases (pt-BR locale).
- **Well-scoped Prisma queries** — Uses `select` for program fields, filters subscriptions to `ACTIVE`, limits transfers to 5 with descending date order, uses `include` to avoid N+1.
- **Pino logging** — Structured log with relevant context (userId, counts). No `console.log` anywhere.
- **Responsive layout** — Grid layouts adapt from single-column mobile to multi-column desktop with `md:` and `lg:` breakpoints.
