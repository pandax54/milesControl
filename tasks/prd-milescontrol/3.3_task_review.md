## Summary

Task 3.3 implements the Pontos Pra Voar scraper extending `BaseScraper`, following the established patterns from tasks 3.1 (Passageiro de Primeira) and 3.2 (Melhores Cartoes). The scraper targets `https://pontospravoar.com/category/promocoes/`, parsing `<article>` elements with `h3.entry-title a` for titles and `a.fancy-categories` for categories. The key differentiator is a dual Portuguese date parser (`parsePtDate`) that handles both relative formats ("2 horas atras") and absolute formats ("19 de marco de 2026"). The implementation is clean, well-tested (43 tests), follows project standards, and is fully consistent with the scraper architecture established in prior tasks.

## Positive Highlights

1. **Robust dual date parser** -- `parsePtDate` cleanly handles two distinct formats: relative Portuguese dates ("X horas/minutos/dias atras") and absolute Portuguese dates ("19 de marco de 2026"). The implementation uses a well-structured `RELATIVE_TIME_PATTERNS` array with typed `unit` discriminants, avoiding a chain of if/else blocks. The `PT_MONTHS` lookup table maps all 12 Portuguese month names with proper accented character support via the `[\wA-u]+` character class in the regex.

2. **Consistent scraper architecture** -- The scraper follows the exact same structural pattern as PdP and MC: constructor with `Partial<ScraperConfig>` overrides, `extractPromotions` override returning `ScrapedPromotion[]`, site-specific helpers exported individually, and structured Pino logging with `{ articlesFound }`.

3. **Full reuse of shared promotion helpers** -- Correctly imports and uses `classifyPromoType`, `extractBonusPercent`, `extractDiscountPercent`, and `extractPrograms` from `./promotion-helpers`, validating the refactoring done in task 3.2.

4. **Pure, individually testable extraction functions** -- `extractCategory`, `extractDateText`, `extractRawContent`, and `parsePtDate` are all pure exported functions with their own dedicated test suites (4, 5, 3, and 12 tests respectively).

5. **Comprehensive test coverage at 100%** -- All 43 tests pass. Statement, function, and line coverage are at 100%. Branch coverage is at 94.11%, with only two defensive fallback branches uncovered (line 78: `attr('href') ?? ''` when href attribute is missing; line 172: `Number.isNaN(date.getTime())` for invalid absolute dates). These are reasonable safety guards.

6. **Well-crafted test fixtures** -- `buildArticleHtml` accurately reproduces the Pontos Pra Voar DOM structure (article > entry-meta > fancy-categories + date links, h3.entry-title > a, entry-excerpt), making integration tests realistic and readable.

7. **Proper use of `vi.useFakeTimers()` for time-dependent tests** -- The relative date parsing test correctly uses `vi.setSystemTime(new Date('2026-03-20T14:00:00'))` and calls `vi.useRealTimers()` in cleanup, following the testing standards.

8. **Clean `extractDateText` implementation** -- Iterates `.entry-meta a` links, skips `fancy-categories` links, and validates text content against known date patterns (`atrás` or `PT_ABSOLUTE_DATE_PATTERN`) before accepting it. Avoids false positives from author links or other meta elements.

9. **Thoughtful `extractRawContent`** -- Combines title and excerpt with a space separator, gracefully handling missing or empty excerpt by returning title only. This provides richer context for downstream promotion classification.

10. **No `any` types, no `console.log`, no `process.env` access** -- Clean TypeScript throughout. All constants use `SCREAMING_SNAKE_CASE`. Readonly arrays with `readonly` modifier on `RELATIVE_TIME_PATTERNS`. Pino logging with structured data.

11. **Singular/plural date unit support** -- The regex patterns handle both "1 hora atras" and "2 horas atras", "1 minuto" and "30 minutos", "1 dia" and "3 dias" via `?` quantifier on the trailing `s`.

12. **Proper `readonly` typing** -- `RELATIVE_TIME_PATTERNS` uses `readonly` on both the array and each element's properties, and the `unit` property uses a union type (`'minutes' | 'hours' | 'days'`) for type safety in `subtractTime`.
