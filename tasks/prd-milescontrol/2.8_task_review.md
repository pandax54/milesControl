### Minor Issues

1. **TooltipProvider wrapping per-link** (both files) -- Each tooltip instance wraps its own `<TooltipProvider>`. The base-ui TooltipProvider manages delay grouping; wrapping each link individually means tooltips in the same row will not share hover delay behavior. This is functionally harmless but slightly redundant. A single `<TooltipProvider>` higher in the component tree (e.g., at the card or layout level) would be more idiomatic. Low priority since it does not affect correctness.

2. **No unit tests for tooltip behavior** -- The tooltip additions are purely presentational React components, and the project does not appear to have component-level tests for UI components in general. Given this is a small UI enhancement with no business logic, the lack of dedicated tests is acceptable, but worth noting for future reference if the project adopts component testing.

## Positive Highlights

- Correct use of `target="_blank"` with `rel="noopener noreferrer"` for security on external links.
