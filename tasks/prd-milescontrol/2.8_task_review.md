## Summary

Task 2.8 required adding direct links to each program's website from the enrollment card, fulfilling PRD requirement F1.12 (inspired by AwardWallet). The implementation enhances pre-existing ExternalLink icon buttons in two components (`enrollment-card.tsx` and `dashboard-balances.tsx`) by wrapping them with shadcn/ui Tooltip components and adding `aria-label` attributes for accessibility. A follow-up fix moved `TooltipProvider` to the dashboard layout for proper delay grouping. The change is small, well-scoped, and correct.

## ✅ Positive Highlights

1. **Security** — Correct use of `target="_blank"` with `rel="noopener noreferrer"` on all external links.
2. **Accessibility** — `aria-label` with dynamic program name provides clear screen reader context (`Open {program name} website`).
3. **Consistent UX** — Tooltip text and pattern are identical across both `enrollment-card.tsx` and `dashboard-balances.tsx`.
4. **Correct render prop** — The `render` prop on `TooltipTrigger` correctly wraps the anchor element, preserving link semantics while enabling tooltip behavior.
5. **Conditional rendering** — `enrollment.program.website &&` guards correctly prevent rendering broken links for programs without websites.
6. **Minimal diff** — No unnecessary changes, focused scope.
7. **TooltipProvider at layout level** — Single `TooltipProvider` in `layout.tsx` ensures all tooltips share consistent hover delay grouping across the entire dashboard.
