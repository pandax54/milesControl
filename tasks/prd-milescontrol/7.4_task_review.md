**2. Non-standard Tailwind class `w-18`**

- **File**: `src/components/promotions/promotion-feed-skeleton.tsx`, line 32
- **Description**: `Skeleton className="h-4 w-18"` — `w-18` is not a standard Tailwind CSS utility. Tailwind's default spacing scale has `w-16` (4rem) and `w-20` (5rem) but not `w-18`. The class is silently ignored, resulting in no explicit width on that skeleton placeholder.
- **Suggestion**: Use an arbitrary value or the closest standard class:

```tsx
// Option A — arbitrary value
<Skeleton className="h-4 w-[4.5rem]" />

// Option B — round to nearest standard
<Skeleton className="h-4 w-16" />
```

## ✅ Positive Highlights

- **Streaming pattern is textbook-correct**: Every page retains auth resolution at the top level then delegates data-fetching to an inner async Server Component wrapped in `Suspense`. This correctly unblocks the initial HTML shell.
- **`DashboardTopNav` streaming**: Moving the `countUnreadNotifications` call behind a Suspense boundary is an excellent optimization — the layout no longer blocks on DB queries before streaming the nav.
- **Parallel data streaming on public page**: Splitting `FeaturedPromotionsSection` and `UpcomingCalendarSection` into sibling Suspense boundaries allows both DB queries to be initiated in parallel rather than sequentially — a meaningful TTFB improvement.
- **`ProgramLogo` fallback chain**: The three-tier hierarchy (bundled local SVG → remote URL with lazy load → initials) is clean and production-ready.
- **NFD normalization**: `normalizeProgramKey` correctly strips diacritics before slug comparison, so `Átomos → atomos` maps correctly to its bundled icon.
- **AVIF + WebP in `next.config.ts`**: Minimal, correct addition — enables Next.js image optimization to serve modern formats with no additional work at call sites.
- **`logoUrl` field propagation**: Properly threaded through `dashboard.service.ts` Prisma select, `EnrollmentSummary` type, and `dashboard-offline-snapshot.ts` Zod schema — no leaky abstraction.
- **Test coverage for new components**: `program-logo.test.tsx` covers all three rendering branches (bundled icon, remote URL, initials). `promotion-feed-skeleton.test.tsx` covers both filter/no-filter variants.
- **Zero `console.log` / no `any` types**: Full compliance with project logging and TypeScript strict-mode rules.
