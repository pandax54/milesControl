### 🟢 Minor Issues

1. **`as never` casts in test mocks** — `potential-balance.service.test.ts` has 12 `as never` casts on `mockFindMany.mockResolvedValue(...)`. Previous reviews (2.4, 2.5, 2.6) consistently flagged and fixed this pattern by using properly typed mock return values. The `buildEnrollment()` factory is well-typed but the wrapping array still needs casting because `findMany`'s return type includes Prisma metadata that the factory doesn't satisfy.

   **Suggested fix**: Type the `buildEnrollment` return to satisfy Prisma's `ProgramEnrollment & { program: ... }` type, or create a type alias matching the `findMany` result shape. This was addressed in prior tasks by importing `ProgramEnrollment` from Prisma and extending the factory accordingly.

## ✅ Positive Highlights

1. **Excellent type guard** — `isValidTransferPartners` properly validates the `Json` field from Prisma with a runtime type guard, handling null, non-array, and malformed entries
2. **Thorough pure function testing** — `parseTransferRatio` and `isValidTransferPartners` are exported and tested independently with edge cases (division by zero, NaN, empty string)
3. **Smart early returns** — The service exits early when there are no airline or banking enrollments, avoiding unnecessary processing
4. **Math correctness** — Uses `Math.floor` for non-integer transfer ratios to avoid fractional miles
5. **Parallel data fetch** — The page uses `Promise.all` to fetch programs, enrollments, and potential balances concurrently
6. **Clean UI** — The card component uses destructuring, shared `formatNumber`, conditional ratio display, and proper dark mode colors
7. **Named constant** — `DEFAULT_TRANSFER_RATIO = 1` avoids a magic number
8. **Proper logging** — Uses Pino with structured context (userId, airlineCount), debug level for empty results, info for computed results
9. **Zero-balance filtering** — Banking programs with 0 balance are correctly excluded from sources
10. **Efficient data access** — Single Prisma query with `include` and `select`, Map lookup for O(n) banking resolution
