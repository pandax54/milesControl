### 🟢 Minor Issues

**1. Unreachable dead code at line 240 in `ratePointPurchase`**

File: `src/lib/services/cost-calculator.service.ts`, line 240

The `return null` at the end of `ratePointPurchase` is unreachable. The early return at line 218 already exits when both `purchasePricePerK` and `purchaseDiscount` are null. The two subsequent `if` blocks (lines 221 and 231) each return when their respective field is non-null. Since at least one must be non-null after passing line 218, line 240 can never execute. This is confirmed by the coverage report showing line 240 as the only uncovered line.

This is a non-blocking issue — the dead code acts as a defensive fallback and TypeScript requires it for the function's return type. However, it could be removed by restructuring:

```typescript
function ratePointPurchase(promotion: {
  purchasePricePerK?: number | null;
  purchaseDiscount?: number | null;
}): CostCalculation | null {
  if (promotion.purchasePricePerK != null) {
    const pricePerPoint = Number(promotion.purchasePricePerK) / POINTS_PER_MILHEIRO;
    return calculateCostPerMilheiro({
      purchasePricePerPoint: pricePerPoint,
      quantity: POINTS_PER_MILHEIRO,
      transferBonusPercent: 0,
    });
  }

  if (promotion.purchaseDiscount != null) {
    const discountedPrice = STANDARD_PRICE_PER_POINT * (1 - promotion.purchaseDiscount);
    return calculateCostPerMilheiro({
      purchasePricePerPoint: discountedPrice,
      quantity: POINTS_PER_MILHEIRO,
      transferBonusPercent: 0,
    });
  }

  return null;
}
```

This removes the early combined-null check and lets the trailing `return null` become reachable (when both fields are null), which is cleaner and would bring coverage to 100%.
