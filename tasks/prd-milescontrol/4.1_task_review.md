### 🟡 Major Issues

**Missing Scraper Integration** — The `processNewPromotions()` function is complete and ready but is **not called** anywhere in the codebase. The scrape-promos cron job (`src/app/api/cron/scrape-promos/route.ts`) calls `runAllScrapers()` and stores promotions but does not trigger alert matching.

- **File**: `src/lib/services/scrape-promos.service.ts`, `src/app/api/cron/scrape-promos/route.ts`
- **Impact**: Alert matching engine exists but will never execute. Users' alerts will not fire when new promotions arrive.
- **Suggested Fix**: After `storePromotions()` completes in `runSingleScraper()`, retrieve the newly stored promotions with their relations and call `processNewPromotions(promotions)`. Alternatively, update `storePromotions()` to accept and process promotions for alerts, or create a separate post-scrape step in the cron route.

Example approach:
```typescript
// In runSingleScraper or scrape-promos route after storage
const storedPromotions = await prisma.promotion.findMany({
  where: { createdAt: { gte: runStartTime } },
  include: { sourceProgram: true, destProgram: true },
});
if (storedPromotions.length > 0) {
  const matchResult = await processNewPromotions(storedPromotions);
  logger.info({ matchResult }, 'Alert matching completed');
}
```
