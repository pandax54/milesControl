### 🟡 Major Issues

**1. List Key Using Array Index — flight-results.tsx:53, 64**

```tsx
// Lines 53, 64 - PROBLEMATIC
cashFlights.map((flight, index) => (
  <CashFlightCard key={`${flight.airline}-${flight.departureTime}-${index}`} flight={flight} />
))
```

Using array index in keys causes React to re-render and lose component state when list order changes. While the composite key with airline/time/index reduces this risk, it's still a code smell.

**Fix**: Add a unique `id` field to `CashFlight` and `AwardFlight` interfaces in the service layer, then use that:

```tsx
cashFlights.map((flight) => (
  <CashFlightCard key={flight.id} flight={flight} />
))
```

### 🟢 Minor Issues

**1. Constant Created Inside Component — award-flight-card.tsx:64**

```tsx
const ratingVariant: Record<PromotionRating, 'default' | 'secondary' | 'outline' | 'destructive'> = {
  EXCELLENT: 'default',
  GOOD: 'secondary',
  ACCEPTABLE: 'outline',
  AVOID: 'destructive',
};
```

Creating this object inside the component (even though it's const) is inefficient. Best practice: move to file-level constant.

**Fix**:
```tsx
// At file level, above component
const RATING_VARIANT_MAP: Record<PromotionRating, 'default' | 'secondary' | 'outline' | 'destructive'> = {
  EXCELLENT: 'default',
  GOOD: 'secondary',
  ACCEPTABLE: 'outline',
  AVOID: 'destructive',
};

// Then in component:
<Badge variant={RATING_VARIANT_MAP[advisorResult.rating]} />
```

**2. Time Formatting Logic Scattered — cash-flight-card.tsx:9-18**

`formatDuration()` and `formatTime()` are defined locally in CashFlightCard but may be needed elsewhere. Consider centralizing in lib/utils/format.ts alongside existing `formatCurrency()` and `formatNumber()`.

**Fix**: Move helpers to `src/lib/utils/format.ts` and export them.

**3. Client-Side Validation Duplicates Schema — flight-search-form.tsx:50-87**

The form's `validate()` function duplicates regex patterns and rules already defined in `flightSearchParamsSchema`. While useful for immediate feedback, this creates a maintenance burden.

**Fix (optional)**: Consider using Zod's `.safeParse()` in the form for validation, or accept this as intentional UX optimization for form feedback.
