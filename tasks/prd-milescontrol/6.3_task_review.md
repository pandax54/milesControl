## ✅ Positive Highlights

1. **Proper Error Handling**: Custom error classes (`ClientNotManagedByAdminError`) distinguish domain-specific errors from generic exceptions. Server actions catch and translate these into user-friendly messages.

2. **Batch Operation Resilience**: `sendBatchRecommendations` gracefully handles per-client failures rather than failing the entire batch. Returns detailed per-client results so callers can inform users of partial successes.

3. **Comprehensive Test Coverage**: 29 new tests covering:
   - Single recommendation success and all error paths
   - Batch recommendations with mixed success/failure scenarios
   - Validation failures (empty clientIds, missing required fields)
   - Authorization checks (authentication, admin role, client ownership)
   - Custom message handling and default body construction

4. **Type Safety**: Full TypeScript strict mode compliance. Result types use `readonly` properties preventing accidental mutations. Zod schemas provide runtime validation with clear error messages.

5. **Admin Ownership Validation**: Both operations verify the admin managing the client before persisting notifications. `findFirst` with `managedById` filter ensures security.

6. **Notification Content Building**: `buildNotificationContent()` fetches promotion details once and reuses for batch operations. Fallback body includes program names and bonus percentages when available.

7. **Proper Logging**: Pino logger used consistently with structured context (adminId, clientId, promotionId, success counts). No `console.log` anywhere.

8. **Clean Component Integration**: Dialogs cleanly separated into single-client and batch variants. `SendRecommendationDialog` appears per-row in client table; `SendBatchRecommendationDialog` appears in footer when 2+ matches exist.

9. **UX Considerations**:
   - Batch dialog shows succeeded/failed summary after send
   - Form closes automatically on success (1200ms delay for confirmation)
   - Optional custom message field with 500 char limit validated by Zod
   - Clear error messages for different failure scenarios

10. **Proper Path Revalidation**: Both server actions revalidate `/admin/promotions` after successful mutations, ensuring UI reflects new notifications.
