## ✅ Positive Highlights

1. **Clean server action architecture**: `calculateCostPerMilheiroAction` and `compareScenariosAction` validate with Zod at the boundary before calling the service layer, use structured error handling with specific error classes (`InsufficientScenariosError`, `TooManyScenariosError`), and log errors with Pino using the correct `{ err: error }` data-first pattern.

2. **Comprehensive test coverage across all layers**:
   - **Action tests** (14 tests): Cover valid input, invalid input (negative values, non-integer quantity), service errors, custom error classes, optional fields, defaults, and invalid scenarios within comparison input.
   - **Service tests** (58+ tests): Cover all pure functions — `calculateTotalMiles`, `calculateTotalCost`, `computeCostPerMilheiro`, `rateByThreshold`, `calculateCostPerMilheiro`, `ratePromotion`, `compareScenarios`, and preset validation.
   - **Schema tests** (19 tests): Cover validation rules, defaults, required/optional fields, edge cases.
   - All tests follow AAA pattern with clear naming and builder functions.

3. **Strong separation of concerns**: Page handles auth/layout, form components handle state/interaction, result card handles display, embed provides compact variant, server actions handle validation/delegation, service layer holds pure business logic, and Zod schemas define the contract.

4. **Well-designed preset system**: Presets displayed as buttons in single mode and clickable badges in compare mode. The `inputToForm`/`parseFormToInput` conversion functions cleanly handle the string-to-number boundary between HTML form state and typed calculator input.

5. **`useTransition` for non-blocking UI**: Both `SingleCalculator` and `CompareCalculator` use React's `useTransition` for server action calls, showing proper pending states ("Calculating..."/"Comparing...") without blocking interaction.

6. **Robust state management in compare mode**: `removeScenario` resets comparison results to prevent index mismatch. `useRef` for ID counter avoids module-level mutable state. Reset function properly restores both state and counter.

7. **Compact embed component (PRD F3.5)**: `PromoCalculatorEmbed` uses `useId()` for globally unique IDs, pre-fills from promotion data, allows quantity/price tweaking, and properly shows error feedback.

8. **Named constants throughout**: `MIN_SCENARIOS`, `MAX_SCENARIOS`, `EMPTY_FORM`, `INITIAL_COUNTER`, `RATING_VARIANTS`, `RATING_LABELS`, and all threshold constants in the service layer. No magic numbers.

9. **Accessibility**: `idPrefix` on `CalculatorFields` ensures unique HTML IDs across multiple scenario instances. `aria-label` on remove buttons provides screen reader context. `htmlFor`/`id` pairs on all labels/inputs.

10. **No prohibited patterns**: Zero `console.log`, zero `any` types, zero direct `process.env` access. All types properly defined with `readonly` on interfaces.
